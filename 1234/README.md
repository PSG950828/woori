# 인프라 매뉴얼 시스템

ZEISS FORUM과 유사한 인터랙티브 인프라 문서화 시스템입니다.

## 📋 프로젝트 개요

- **확장 가능한 구조**: 언제든지 새로운 섹션과 문서를 추가/수정할 수 있습니다
- **직관적인 네비게이션**: 사이드바를 통한 체계적인 목차 구성
- **반응형 디자인**: 다양한 디바이스에서 최적화된 사용 경험
- **검색 기능**: 전체 문서에서 빠른 내용 검색
- **편집 기능**: 웹 인터페이스에서 직접 문서 편집 가능
- **로컬 저장**: 브라우저 로컬 스토리지를 통한 데이터 보존
- **로그인 게이트**: LAN에서 접속할 때 계정 인증을 통과해야만 콘텐츠 열람/편집 가능
- **엑셀 표 붙여넣기**: IP 목록 등 엑셀 데이터를 그대로 붙여넣어 Markdown 표로 자동 변환
- **브랜드 커스터마이징**: `assets/logo.svg`, `assets/welcome-icon.svg` 파일을 교체하면 즉시 맞춤 로고와 이미지를 사용할 수 있습니다

## 🚀 빠른 시작

### 1. 원클릭 시작/중지 (가장 직관적인 방법)
- `tools/StartServer.cmd`: 더블클릭만 하면 포트 5173으로 Node 서버가 켜집니다.
- `tools/StopServer.cmd`: 현재 실행 중인 서버를 즉시 중지합니다.
- `tools/ServerControlGUI.cmd`: 앞서 만든 WinForms 컨트롤러(`ServerControl.ps1`)를 GUI 모드로 열어서 버튼으로 조작할 수 있습니다.

> 팁: 위 세 파일을 바탕화면으로 끌어다 놓으면 자동으로 바로가기(.lnk)가 만들어지므로 “서버 시작 / 서버 중지” 아이콘처럼 사용할 수 있습니다.

### 2. Infra Manual GUI 컨트롤러
PowerShell 실행 정책을 한 번만 아래처럼 허용한 뒤 `tools/ServerControl.ps1`을 실행하면 포트 입력, 로그, 브라우저 열기까지 한 화면에서 제어할 수 있습니다.

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 3. 간단한 정적 서버

```bash
python -m http.server 8000
# 또는
npx serve .
```

### 4. 직접 실행
`index.html` 파일을 웹 브라우저에서 직접 열어서 사용할 수 있습니다.

### 5. 자체 서버(IIS) 배포
1. Windows 서버에 저장소를 복사하고 관리자 PowerShell에서 `deploy\self-host` 로 이동합니다.
2. 아래 명령으로 IIS + 방화벽 + 사이트 구성을 자동화합니다.
	```powershell
	./setup.ps1 -SiteName "InfraManual" -Port 8080 -PhysicalPath "C:\inetpub\infrastructure-manual"
	```
3. 배포 후 `http://<서버 IP>:8080`으로 접속하면 어디서든 동일한 브라우저 UI로 열람할 수 있습니다.
4. HTTPS 바인딩, IP 제한 등 추가 설정 방법은 `deploy/self-host/README.md`를 참고하세요.

## 📖 사용 방법

### 기본 네비게이션
- **사이드바**: 왼쪽 사이드바에서 카테고리를 클릭하여 펼치기/접기
- **문서 보기**: 세부 항목을 클릭하여 해당 문서 내용 확인
- **이전/다음**: 하단의 네비게이션 버튼으로 순차적 탐색

### 문서 편집
1. 문서를 열고 상단의 **편집** 버튼 클릭
2. 관리자 잠금이 설정된 경우 비밀번호를 입력해 잠금을 해제합니다
3. Markdown 형식으로 내용 수정
4. **저장** 버튼으로 변경사항 적용

### 로그인 및 계정 관리
- 웹 페이지에 접속하면 먼저 **로그인 화면**이 표시됩니다. 최초 기본 계정은 `admin / woori0212!` 입니다.
- 필요하다면 서버 실행 전에 `DEFAULT_ADMIN_USER`, `DEFAULT_ADMIN_PASSWORD` 환경 변수를 지정해 초기 계정을 커스터마이징할 수 있습니다.
- 로그인에 성공하면 본문이 즉시 열리고, 헤더 우측에 **계정 관리**·**로그아웃** 버튼이 활성화됩니다.
- **계정 관리** 버튼을 누르면 현재 등록된 계정 목록을 확인하고 아이디 변경 / 비밀번호 재설정 / 삭제를 수행할 수 있습니다.
	- 새 계정을 추가할 때는 아이디와 6자 이상의 비밀번호를 입력한 뒤 `계정 생성`을 선택합니다.
	- 기존 계정의 아이디/비밀번호를 변경하거나 삭제하면 즉시 서버의 SQLite DB에 반영되며, 마지막 한 개 계정은 안전을 위해 삭제할 수 없습니다.
- 인증 토큰이 만료되거나 로그아웃하면 다시 로그인 화면으로 전환되며, 재로그인 시 서버 상태가 자동으로 동기화됩니다.

### 관리자 잠금
- 상단 도구 모음의 **관리자 잠금** 버튼으로 편집/삭제 기능을 보호할 수 있습니다
- 처음 누르면 비밀번호 설정 모드가 열리며, 이후에는 잠금 해제 → 다시 잠그기 → 비활성화 동작을 제공합니다
- 잠금 해제 시 한 번 인증하면 계속 편집할 수 있으며, 다시 잠그기를 실행하면 모든 보호 작업이 즉시 잠깁니다
- 비밀번호 변경 또는 잠금 비활성화는 관리자 모달의 옵션을 통해 실행합니다
- **콘텐츠 관리** 패널도 동일한 비밀번호로 보호됩니다. 잠금이 설정되지 않았다면 먼저 비밀번호를 등록해야 하며, 잠긴 상태에서는 인증 후에만 패널이 열립니다.

### 엑셀 데이터 붙여넣기
1. 문서를 연 뒤 상단의 **엑셀 표 붙여넣기** 버튼 클릭 (필요하면 자동으로 편집창이 열립니다)
2. 엑셀에서 원하는 범위를 복사(Ctrl+C) 후 모달 입력창에 붙여넣기
3. 미리보기 영역에서 자동 생성된 Markdown 표를 확인
4. **문서에 삽입**을 누르면 커서 위치에 표가 추가됩니다 (첫 행을 머리글로 사용할지 체크 박스로 선택 가능)

#### Word/Excel 콘텐츠를 그대로 붙여넣고 싶다면
- 편집 모달의 입력 영역에 곧바로 **Ctrl+V** 하면 Word/Excel 특유의 Mso 스타일을 자동으로 정리하고 표/목록 구조는 그대로 유지합니다.
- 붙여넣기가 발생하면 커서 근처에 **붙여넣기 옵션 팝오버**가 뜨며, "서식 유지", "텍스트만", "표로 변환" 가운데 원하는 방식을 골라 삽입할 수 있습니다.
- 우측 **붙여넣기 미리보기 패널**에서 원본/결과를 즉시 비교하고, 모드를 바꾸거나 HTML·텍스트를 복사하거나 미리보기 내용을 그대로 삽입할 수 있습니다 (더블클릭/드래그 지원).
- 다중 표가 감지되면 표 리스트가 나타나 선택한 표만 모아 삽입·복사하거나 각 표를 개별 미리보기로 열어볼 수 있습니다.
- 붙여넣기 설정(유지할 서식, Word 스타일 매핑, 기본 모드)을 수정하면 현재 클립보드 미리보기가 즉시 갱신되어 결과를 바로 확인할 수 있습니다.
- 탭과 줄바꿈으로 구성된 엑셀 텍스트를 붙여넣으면 첫 행을 헤더로 인식해 HTML 표로 자동 변환됩니다.
- 기존처럼 모달을 이용해 붙여넣기 전 미리 표 구조를 확인할 수도 있습니다 (둘 중 편한 방법 사용).

#### 붙여넣기 미리보기 패널 한눈에 보기
1. **원본/적용 결과 비교**: 좌측에는 클립보드 원본, 우측에는 선택한 모드의 결과가 표시됩니다.
2. **모드 전환**: 드롭다운에서 서식 유지/텍스트만/표/원본 HTML을 즉시 바꿀 수 있습니다.
3. **표 선택 컨트롤**: 표가 여러 개일 경우 리스트에서 필요한 표만 선택·미리보기·삽입·복사할 수 있습니다.
4. **미리보기 삽입 & 복사**: 버튼 한 번 또는 더블클릭/드래그로 결과를 문서에 삽입하거나 클립보드로 복사합니다.
5. **설정 바로가기**: 패널에서 곧바로 붙여넣기 규칙 모달을 열어 서식 유지 규칙과 Word 스타일 매핑을 관리할 수 있습니다.

브라우저 콘솔에서:
```javascript

resetData(); // 모든 데이터 초기화
```

```powershell
cmd /c npm run snapshot
```

### 1. ZEISS FORUM 스타일 GUI 컨트롤러 (추천)
1. `tools/ServerControl.ps1` 파일을 우클릭 후 **PowerShell에서 실행**을 선택합니다. (처음 실행 시 `Set-ExecutionPolicy -Scope CurrentUser RemoteSigned` 를 한 번만 수행하면 됩니다.)

```powershell
powershell -ExecutionPolicy Bypass -File .\tools\ServerControl.ps1 -Command stop
```

## 🔐 보안 고려사항

- 로컬 스토리지만 사용 (외부 서버로 데이터 전송 없음)
- XSS 방지를 위한 콘텐츠 이스케이핑
- 클라이언트 사이드에서만 동작
- Express 서버는 `/api/auth/*`, `/api/accounts`, `/api/state` 모든 엔드포인트에 JWT 기반 인증을 적용했습니다.
- 기본 계정(`admin / woori0212!`)은 최초 접속 후 바로 교체하거나 새 계정을 추가하는 것을 권장합니다.
- 엑셀 붙여넣기 기능 또한 브라우저 메모리 내에서만 처리되어 외부로 전송되지 않습니다
- 파괴적 작업(편집, 삭제, 구조 변경)은 **관리자 잠금**을 통해 비밀번호로 보호할 수 있으며, 설정 정보 역시 로컬에만 저장됩니다
